  2. 工具 Schema 构建和发送

  4. 模型决策过程

  模型接收到这些信息后：

  1. 分析用户需求: "分析项目代码结构，找出 React 组件文件"
  2. 扫描可用工具: 看到有 Glob(文件搜索)、View(文件读取)等工具
  3. 选择最佳工具组合:
    - 用 Glob 工具搜索 **/*.tsx 文件
    - 可能用 View 工具读取找到的组件文件

  6. 递归执行

  系统执行工具后，会将工具结果再次发送给模型，进行递归查询：

  // src/query.ts:331-338
  yield* await query(
    [...messages, assistantMessage, ...orderedToolResults], // ← 包含工具结果的完整对话
    systemPrompt,
    context,
    canUseTool,
    toolUseContext,
    getBinaryFeedbackResponse,
  )

  关键特点

  1. 全量工具信息: 系统会将所有启用的工具的 Schema 和描述都发送给模型
  2. 智能选择: 模型根据任务需求自主选择合适的工具
  3. 递归对话: 工具执行结果会加入对话历史，模型可以基于结果继续决策
  4. 工具链: 模型可以组合多个工具完成复杂任务
